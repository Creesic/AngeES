# UI Update: Port the 1.14a menu (Load Engine / Reload / Load Theme / Exit + Console [F5])

**Branch:** `UI-update`
**Date:** 2026-07-15
**Status:** Design approved (pending spec review)

## Goal

Bring the extra UI controls from the binary-only `engine-sim v0.1.14a` build into the
current source tree:

- A **top button bar**: `Load Engine`, `Reload`, `Load Theme`, `Exit`.
- A **named-view system with a clickable tab bar** and per-view keyboard shortcuts,
  including a `Console [F5]` view — replacing the current ad-hoc `Tab`-cycle screen switch.

1.14a is a later build of *this same codebase*, so this is a "port the additions" effort,
not a new UI framework. The same `UiElement` / `UiButton` / `UiManager` classes already
exist in the tree; 1.14a extended `EngineSimApplication` and added two `UiElement`
subclasses.

## Baseline vs. 1.14a delta

| Capability | Current source | 1.14a |
|---|---|---|
| Reload script | `Enter` key → `loadScript()` (src/engine_sim_application.cpp:381) | GUI button |
| Screen switch | `Tab` cycles `m_screen` 0/1/2, three hardcoded layouts (l.389, l.969/1002/1017) | Named views + clickable tab bar + per-view keys (F5) |
| Load arbitrary engine `.mr` | — | `Load Engine` button + file dialog |
| Load theme | — (colors hardcoded, l.167-178 of the header) | `Load Theme` button |
| Quit | window-close only | `Exit` button |

## Decisions (locked)

- **File picker:** full native on both platforms — Win32 `GetOpenFileName` on Windows,
  `NSOpenPanel` on macOS, behind a `PlatformFileDialog` seam (matching the port's existing
  `#if defined(_WIN32)` / `defined(__APPLE__)` gating).
- **Console [F5]:** replicate the 1.14a named-view switcher (approach **B2** below).
- **Load Theme:** include the button for visual parity but leave it **disabled / no-op**
  for now (the general theming system is out of scope for this change).

## Architecture

### Component 1 — Toolbar cluster (`UiToolbar`)

New `UiElement` subclass mirroring the existing clusters (`InfoCluster`, etc.). Owns four
`UiButton` children (`Load Engine`, `Reload`, `Load Theme`, `Exit`), font size `16`, laid
across a top strip via the existing `Grid` / `Bounds` helpers. Overrides
`signal(UiElement *element, Event::Clicked)` to dispatch by child — the exact pattern
reverse-engineered from 1.14a's `sub_14001CCF0`:

- `Load Engine` → `app->requestEngineReload(PlatformFileDialog::openFile(...))`
- `Reload` → `app->requestEngineReload(currentScriptPath)`
- `Load Theme` → disabled (stub; button rendered but takes no action)
- `Exit` → `app->quit()`

Registered under `m_uiManager` root and given bounds in the per-view layout.

### Component 2 — Deferred reload + cross-platform file dialog

- Generalize `EngineSimApplication::loadScript()` → `loadScript(const std::string &path)`
  so `Load Engine` can load any `.mr` and `Reload` re-runs the current one, reusing the
  existing stop-audio → load → restart-audio sequence (l.381-387).
- Add a pending-request field on the app (`std::string m_pendingScriptPath;
  bool m_reloadRequested;`), applied at the top of `process(dt)`. This reproduces 1.14a's
  **deferred** reload (`EngineSim_QueueEngineScriptReload`) — never tear down simulation
  objects inside a click/signal callback.
- Track the currently loaded script path so `Reload` knows what to re-run.
- New `PlatformFileDialog` seam:
  - `platform_file_dialog.h` — `std::optional<std::string> openFile(const char *title, const char *filter);`
  - `platform_file_dialog.cpp` — Win32 `GetOpenFileName` (COMDLG32), gated `#if defined(_WIN32)`.
  - `platform_file_dialog.mm` — `NSOpenPanel`, gated `#if defined(__APPLE__)`; added to the
    macOS source list in CMake.
- Bracket the modal dialog with input pause/resume (1.14a does this via
  `sub_14000DBA0` / `sub_140011CB0`) so a held mouse button doesn't leak into the sim.

### Component 3 — Exit

`EngineSimApplication::quit()` closes the delta-studio game window / ends the `run()` loop
(reusing delta-studio's window-close path). The `Exit` button and, optionally, a keybind
call it.

### Component 4 — View registry + tab bar + Console [F5] (approach B2)

Refactor the ad-hoc `m_screen` (0/1/2, cycled by `Tab`) into a small **named-view registry**.
Each view is data:

```
struct View {
    std::string name;        // e.g. "Console"
    std::string keyLabel;    // e.g. "F5"  -> rendered as "Console [F5]"
    ysKey::Code shortcut;    // key that selects this view
    // layout / visibility configuration for the clusters in this view
};
```

- The existing three layouts (full dashboard / engine-only / partial) become named views.
- Add a **Console** view (shortcut `F5`).
- New `UiTabBar` (`UiElement` subclass) renders each registered view as a clickable tab
  showing `"Name [Key]"` (literally `name + " [" + keyLabel + "]"`, matching the binary's
  `"Console"` + `" [F5]"` concatenation), highlights the active view, and switches on click.
- Per-view shortcut keys bound in `processEngineInput` (F5 → Console). `Tab` is retained as
  "cycle to next view" for backward compatibility.
- The Console view surfaces the delta-basic console (`m_engine.GetConsole()`, already wired
  up in the tree). Exact contents to be confirmed by further RE (see Open Items).

## Success criteria

- Windows build (MSVC, the tree's primary target) compiles and links with the new UI.
- The four toolbar buttons appear and: `Load Engine` opens a native picker and loads the
  chosen `.mr`; `Reload` re-runs the current script; `Exit` closes the app; `Load Theme`
  is visibly present but inert.
- Views are selectable by clicking tabs and by shortcut key; `Console [F5]` selects the
  Console view; the active tab is visually indicated.
- No regression to the existing `Enter`/`F` key behavior or the simulation.

## Appendix — 1.14a reverse-engineering reference

Binary: `engine-sim-v0.1.14a/bin/engine-sim-app.exe`, imagebase `0x140000000`.

**Toolbar cluster** (vtable `0x1400c82a8`):
- `sub_14001C730` — layout/label method; sets 4 buttons at object offsets +112/+120/+128/+136
  with labels `Load Engine` / `Reload` / `Load Theme` / `Exit`, `m_fontSize = 16.0`.
- `sub_14001CCF0` (`EngineSim_HandleScriptLoadUiAction`) — `signal()` click dispatcher:
  - `+112` Load Engine → pause (`sub_14000DBA0`) → `EngineSim_OpenMrScriptFileDialog`
    (`GetOpenFileNameA`, COMDLG32 `0x1400c6000`) → resume (`sub_140011CB0`) →
    `EngineSim_QueueEngineScriptReload`.
  - `+120` Reload → `EngineSim_QueueEngineScriptReload(WindowName)`.
  - `+128` Load Theme → dialog → `EngineSim_QueueThemeScriptReload`.
  - `+136` Exit → `sub_1400113C0` (quit).

**View manager** (vtable `0x1400c7020`):
- `sub_140003750` — builds view clusters + registry; registers
  `{cluster, "Engine Visualization", "Engine"}` and `{cluster, "Console", "Console"}`.
  View struct is `0x48` bytes: `{UiElement*, std::string longName, std::string shortName}`,
  stored in a vector at object offsets [18..20].
- Other virtuals `sub_1400035c0` / `sub_140003fc0` / `sub_140003ad0` / `sub_140003f70` —
  tab render / view switch / input (to RE during implementation).
- Strings: `"Console"` `@0x1400c70a0`, `" [F5]"` `@0x1400c70a8`.
